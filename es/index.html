⚡ Por qué estos tres juntos
Cubren los tres patrones que sostienen los productos de IA/LLM modernos: memoria aumentada por recuperación (RAG con hechos del usuario persistentes), orquestación multi-agente en paralelo con salidas estructuradas (Promise.all + Zod vía generateObject), y bucles autónomos de uso de tools con terminación limpia (ToolLoopAgent + tool terminal sin execute). Construidos de punta a punta en TypeScript sobre Next.js 16 + Vercel AI SDK v6, desplegados en Vercel, con READMEs lo bastante detallados como para poner al día a otro ingeniero en una tarde.

AdminRent — Gestión de propiedades residenciales 🇵🇾

Personal · privado

Proyecto personal construido de punta a punta en mi tiempo libre. SaaS multi-tenant para gestionar complejos residenciales: los admins manejan propiedades, inquilinos y cobro de alquileres desde un panel web; los residentes pagan, suben comprobantes y reciben notificaciones push desde una app móvil; las facturas se emiten a través de la autoridad tributaria SIFEN de Paraguay vía un microservicio independiente.

Arquitectura · 4 servicios
  • web Panel de admin en Next.js 16 — App Router, Server Components, route groups (auth) / (dashboard), Supabase SSR con JWT refrescado dentro de un proxy/middleware de Next.js que además protege las rutas con auth.
  • api API REST en Flask 3 con arquitectura por capas (api / services / models / middleware) sobre PostgreSQL, con migraciones Alembic aplicadas automáticamente en cada release de Railway.
  • mobile App Flutter para residentes con estado Riverpod, Dio HTTP, Firebase Auth + push FCM, y sqflite como caché offline para escenarios de baja conectividad.
  • sifen Microservicio independiente en TypeScript + Express que firma y envía facturas XML por SOAP a la autoridad tributaria de Paraguay: base de datos propia, cron jobs e imagen Docker.
Stack completo
Next.js 16 · React 19 Flask 3 · SQLAlchemy 2 Flutter · Riverpod Express · TypeScript (SIFEN) PostgreSQL · Supabase Railway · Docker
